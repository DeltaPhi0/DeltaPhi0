**Core Architecture & Security**:
I recently migrated from a Raspberry Pi to an ex-gaming PC as my primary server, implementing a highly secure remote-boot workflow:
- **Remote Access & Boot**: I connect through a secondary lightweight server acting strictly as a VPN gateway. From there, I send Magic Packets (Wake-on-LAN) to trigger the main server's boot sequence.
- **Full Disk Encryption**: The main server utilizes LUKS encryption. Upon boot, I SSH into a Dropbear initramfs environment to manually input the LUKS passphrase and unlock the system.
- **Network Management**: Local network requests are routed through a personal DNS server (Pi-hole) and managed via Nginx Proxy Manager.
- **Automated Backups**: A custom automated service archives critical server data into `tar.gz` files and syncs them directly to Google Drive.

**Containerized Stack (Docker)**:
My server runs a robust, fully containerized ecosystem managed via Portainer:
- **Media & Content**: Radarr, Sonarr, Lidarr, Bazarr, Prowlarr, qBittorrent, Flaresolverr, Jellyfin(Movies & TV shows) Jellyseerr, Navidrome (Music), Audiobookshelf, and Kavita(Books).
- **Data & Productivity**: Immich (Machine-learning powered photo backups), Trilium (Notes), Anki Sync Server, and Code-Server (Web IDE).
- **Monitoring & Management**: Homarr (Dashboard), Dashdot, Uptime Kuma, Dozzle (Logs), Watchtower (Auto updates), and Deunhealth.
- **Custom Deployments**: 4gaboards (App + PostgreSQL database).
